## 2.2 Design Principles

| Principle | Implementation |
|-----------|----------------|
| **Separation of Concerns** | Data (JSON) is never mixed with presentation (HTML) |
| **Single Source of Truth** | Each content type has ONE JSON file |
| **Content-First** | Design adapts to content, not vice versa |
| **Mobile-First** | All layouts designed for mobile, then enhanced |
| **Bengali Typography** | Proper font loading, line-height, readability |
| **Progressive Enhancement** | Works without JS (basic), enhanced with JS |
| **Semantic HTML** | Proper heading hierarchy, landmarks, accessibility |
